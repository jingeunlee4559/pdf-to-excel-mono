아래 순서대로 확인하여 첫 번째 해당 유형 적용:

① 보고서 양식 요청 확인
  키워드: "검토보고서", "업무보고서", "내부보고", "보고서 형식", "보고서로", "보고서처럼", "결재용"
  → tableType="NARRATIVE_REPORT", rows=[], columns=[]
  → 가격 데이터 생성 금지, 서술형 보고서는 report_generator가 별도 처리

② 회의록 양식 요청 확인
  키워드: "회의록으로", "회의록 형식", "회의록처럼"
  → tableType="MEETING_MINUTES"
  → 컬럼: [{key:"agenda_no",label:"안건번호"},{key:"agenda_title",label:"안건명"},{key:"discussion",label:"토의내용"},{key:"decision",label:"결정사항"},{key:"department",label:"담당부서"},{key:"assignee",label:"담당자"},{key:"due_date",label:"완료기한"},{key:"remark",label:"비고"}]

③ 공문 양식 요청 확인
  키워드: "공문으로", "공문 형식", "행정문서로"
  → tableType="OFFICIAL_LETTER"
  → 컬럼: [{key:"section",label:"항목"},{key:"content",label:"내용"},{key:"remark",label:"비고"}]

④ 업체 비교표 요청 확인
  키워드: "업체별", "회사별", "비교표로", "단가비교"
  → tableType="VENDOR_PRICE_COMPARISON"
  → 각 업체별 단가/금액 컬럼 동적 생성

⑤ 원문 자동 판별
  위 키워드 없음 → 원문 내용 기반으로 가장 적합한 tableType 자동 선택

[VENDOR_PRICE_COMPARISON] 업체별 단가 비교표
  필수 컬럼: NO(순번), item_name(품목명), spec(규격), quantity(수량), unit(단위), std_price(표준단가)
  업체별 반복: {업체명}_unit_price(단가), {업체명}_amount(금액)
  후미 컬럼: lowest_vendor(최저업체), lowest_price(최저단가), remark(비고)
  예시행: {"no":1,"item_name":"고압배전반 설치","spec":"3.3kV 급, 4","quantity":30,"unit":"개","std_price":285600,"한국전기_unit_price":257757,"한국전기_amount":7732710}

[SINGLE_ESTIMATE] 단일 견적서
  필수 컬럼: no(NO), item_name(품목명), spec(규격), quantity(수량), unit(단위), unit_price(단가), amount(금액), remark(비고)

[MEETING_MINUTES] 회의록
  필수 컬럼: agenda_no(안건번호), agenda_title(안건명), discussion(토의내용), decision(결정사항), department(담당부서), assignee(담당자), due_date(완료기한), remark(비고)

[OFFICIAL_LETTER] 공문
  필수 컬럼: section(항목), content(내용), remark(비고)

[CONTRACT] 계약서
  필수 컬럼: clause_no(조항번호), clause_title(조항명), content(내용), remark(비고)

[INSPECTION_REPORT] 점검보고서
  필수 컬럼: no(NO), check_item(점검항목), result(점검결과), judgment(판정), defect(불량내용), action(조치사항), inspector(점검자), check_date(점검일)

[PROGRESS_REPORT] 공정보고서
  필수 컬럼: work_type(공종명), planned_rate(계획공정률), actual_rate(실적공정률), diff(차이), delay_reason(지연사유), action_plan(조치계획)

[PURCHASE_ORDER] 발주서
  필수 컬럼: no(NO), item_name(품목명), spec(규격), quantity(수량), unit(단위), due_date(납기일), remark(비고)

[STANDARD_MARKET_PRICE] 표준시장단가표
  필수 컬럼: code(공종코드), work_name(공종명), spec(규격), unit(단위), std_price(표준단가), basis(적용기준), source_page(출처), remark(비고)

[GENERAL_BUSINESS] 일반 업무표
  원문에서 가장 의미 있는 구조를 자동으로 설계 (최소 3컬럼, 최대 15컬럼)

[없는 데이터 처리 — 가장 중요]
- 원문에 없는 값 → null (절대 만들어내지 않음)
- 원문에 있으나 읽기 불가 → "원문 미기재"
- 부분적으로 확인 가능 → 확인된 값 + "(일부)"
- OCR 오류 의심 → 그대로 넣고 issue 추가

[숫자 파싱]
- 쉼표 포함 숫자 → number 타입으로 변환 (1,234,567 → 1234567)
- "원"/"만원"/"억원" 단위 변환 후 number 타입
- 퍼센트(%) → 원본 그대로 string 유지 ("15%")
- 수량×단가와 금액 차이 5% 초과 → AMOUNT_MISMATCH issue

[단위 정규화]
- EA/PCS/개 → "개"
- m2/M2/평방미터 → "㎡"
- m3/M3/세제곱미터 → "㎥"
- 공m3 → "공㎥"
- hr/H → "시간"
- 원본 단위는 unit_original에 보존

[업체명 정규화]
- ㈜/주식회사/(주) 차이 무시, 동일 업체로 처리
- 표기 통일: 앞에 "㈜" 형식으로
