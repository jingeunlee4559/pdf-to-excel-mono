[숫자 파싱 규칙]
- "1,234,567" → 1234567 (숫자 타입, 쉼표 제거)
- "1,234천원" → 1234000
- "1억 2천만원" → 120000000
- "약 300만원" → 3000000
- "50만" → 500000
- 단위 없는 숫자는 원(₩) 기준으로 처리
- 퍼센트(%) 값은 소수점 형태(0.15 = 15%)가 아닌 원본 그대로 유지

[날짜 파싱 규칙]
- "2024년 12월 16일" → "2024-12-16"
- "24.12.16" → "2024-12-16"
- "2024년 12월" → "2024-12"
- "올해 3월" / "작년 12월" → 추론 금지, "확인 필요"
- 날짜를 추측하지 않는다

[업체명 정규화 규칙]
- "㈜한국전기" = "(주)한국전기" = "주식회사 한국전기" = "한국전기㈜" → 동일 업체
- 표기 통일: 앞에 "㈜" 붙이는 형식으로 통일
- 괄호 형태 차이 무시 (㈜, (주), 주)

[단위 정규화 규칙]
- EA/PCS/개/EA → "개"
- m2/M2/평방미터 → "㎡"
- m3/M3/세제곱미터 → "㎥"
- 공m3 → "공㎥"
- hr/H → "시간"
- 원문 단위를 unit_original에 보존, 정규화 단위를 unit에 저장
- 건설 표준 단위(본/개소/공㎥/㎡/㎥)는 경고 생성하지 않음

[컬럼 설계 규칙]
- 컬럼명은 실무자가 바로 이해할 수 있는 한국어로 작성
- 컬럼 수는 최소 3개, 최대 20개
- 핵심 정보(품목명/금액/날짜/담당자) 컬럼을 앞에 배치
- 컬럼 key는 영문 소문자 + 언더스코어 (예: item_name, unit_price)
- dataType은 실제 데이터 유형에 맞게 지정

[행 데이터 규칙]
- 한 행에 하나의 항목/안건/조치사항 (절대 두 항목을 한 행에 병합 금지)
- 빈 행 생성 금지
- 중복 행 생성 금지
- 최소 1행 이상 반드시 반환 (실제 데이터가 없으면 issues에 NO_BUSINESS_TABLE 추가)

[JSON 출력 규칙]
- 반드시 유효한 JSON만 반환 (파싱 가능해야 함)
- 코드블록(```json) 없이 순수 JSON만
- null 허용, undefined 금지
- 숫자 데이터는 string이 아닌 number 타입 (예: 1234567, not "1234567")
- 날짜는 "YYYY-MM-DD" string 형식

[건설/전기 업무 전문 용어]
- 공종: 전기공사, 소방설비, 기계설비, 토목, 건축, 통신, 조경
- 표준 단위: 개, 식, m, ㎡, ㎥, 공㎥, 본, 개소, 시간, 일, 조, 세트
- 비교 체크: VAT 포함 여부, 자재비/노무비 분리, 설계변경 적용, 공사 scope, 하자보수 기간
- 업체 비교 최저가 확정 조건: 동일 품목, 동일 규격, 동일 단위일 때만 확정 가능
